## Журнал

### BUG-001 — После обновления страницы показывается «Участник» вместо имени/роли ✅ Закрыт

**Эпик:** A2 (A2.3) / S1  
**Причина:** `roleName` не возвращался из login (не было в `UserResponseSchema` Zod); `formatUser` не вызывался в сервисе  
**Фикс:** добавлен `roleName` в `UserResponseSchema`; `login` возвращает `formatUser(user)` с `include: { orgRole }` из Prisma; шапка показывает `user.roleName ?? 'Участник'`  
**Статус:** ✅ Закрыт (подтверждено пользователем 2026-06-24)

### BUG-003 — Страница «Аналитика» недоступна из навигации 🔴 Открыт → исправлен

**Эпик:** A5  
**Шаги воспроизведения:** открыть сайдбар как admin — пункта «Аналитика» нет  
**Ожидалось:** ссылка на `/dashboard/analytics` в сайдбаре (adminOnly)  
**Фактически:** страница существует, но попасть на неё можно только по прямому URL  
**Причина:** в `Sidebar.tsx` в секции «Управление» отсутствует пункт «Аналитика»  
**Фикс:** добавлен пункт в `SECTIONS` Sidebar.tsx  
**Статус:** ✅ Закрыт
**Фикс:** `authStore.ts` — `setAuth` сохраняет `user` в localStorage, `clearAuth` удаляет; инициализация читает оба поля  
**Статус:** ✅ Закрыт (подтверждено пользователем)

### BUG-004 — Белый экран на странице «Создать проект» 🔴 Открыт → исправлен

**Эпик:** PM1  
**Шаги воспроизведения:** Проекты → «Создать проект»  
**Ошибка:** `users?.map is not a function` (CreateProjectPage.tsx:151)  
**Причина:** `GET /api/users` возвращает `{ items: [], total, page, pages }`, а код делал `r.data` напрямую как массив  
**Фикс:** `r.data.items ?? r.data` в queryFn  
**Статус:** ✅ Закрыт

### BUG-006 — WBS: ячейка меняется при фокусе без редактирования ✅ Закрыт

**Эпик:** PM2 Планировщик  
**Шаги воспроизведения:** открыть планировщик → кликнуть на ячейку «Длит.» или «Финиш» любой задачи → сразу кликнуть в другое место без ввода  
**Ожидалось:** строка не подсвечивается, значения не меняются  
**Фактически:** строка подсвечивается янтарным (pending change), значения задачи пересчитываются как будто было внесено изменение  
**Причина:** `EditableCell.onBlur` вызывал `onCommit` безусловно, даже когда значение не изменилось  
**Фикс:** добавлен `originalVal` ref; `onCommit` вызывается только если `e.target.value !== originalVal.current`  
**Файл:** `frontend/src/pages/gantt/GanttGrid.tsx`, компонент `EditableCell`  
**Статус:** ✅ Закрыт (подтверждено пользователем 2026-06-29)
